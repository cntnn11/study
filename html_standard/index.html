⇒display:inline-block 间隙问题

原因：有空格或换行

解决：删除元素后的空格或换行

⇒height 和 line-height 要大于 font-size 至少 2px ,否则英文小写 q , g , y 会显示不完整（firefox）

⇒不要给 <th> 或 <td> 设置 position:relative 因为firefox不支持,不过可以嵌套 <div> 来解决

⇒IE 6\7 不支持 <th> 设置 border 样式

⇒IE6,7 下<textarea>不支持 height:100%

⇒给 <table> 设置 table-layout:fixed 和 width 可以解决 <table> 自动伸缩问题

⇒IE6\7 下 overflow:hidden 不隐藏

原因：可能是子元素有 position:relative

解决：给最外层同时也设置 position:relative

⇒IE6,7块元素不支持 display:inline-block

原因：未触发 hoslayout

解决：同时设置 *display:inline 和 *zoom:1

⇒!important 在IE 6 下不能在同一行样式中使用,否则不生效

⇒栅格化概念

栅格系统英文为"grid systems"，开始是从平面栅格系统中发展而来，以规则的网格阵列来指导和规范网页中的版面布局以及信息分布 。

通俗点讲是一种平面设计的方法与风格，运用固定的格子设计版面布局，其风格工整简洁。

⇒栅格化的优势

  • 规范性，规律性——易于维护，节约成本
  • 一致性——提升用户体验
  • 灵活性——便于整体布局和区域设计
  • 模块化——便于模块化设计，重复利用

⇒栅格化原理与运用

⇒语义化概念

首先是关于语义（Semantics）和默认样式的区别，默认样式是浏览器设定的一些常用tag的表现形式，个人认为他的主要目的就是让大家直观的认识标签(markup)和属性(attribute)的用途和作用，很明显Hx系列看起来很像标题，因为拥有粗体和较大的字号。<strong>,<em>用来区别于其他文字，起到了强调的作用。至于列表和表格很明显的告诉你他们是做什么的。

其次，语义化的网页的好处，最主要的就是对搜索引擎友好，又了良好的结构和语义你的网页内容自然容易被搜索引擎抓取，你网站的推广便可以省下不少的功夫。

⇒部分标签语义

写语义化的 HTML 结构其实很简单，首先掌握 HTML 中各个标签的语义，在看到内容的时候想想用什么标签能更好的描述它，是什么就用什么标签。

  • <h1>、<h2>、<h3>、<h4>、<h5>、<h6> 作为标题使用，并且依据重要性递减
  • <p> 段落标记，<p> 中的文字会自动换行，而且换行的效果优于 <br/>
  • <ul>、<li> 无序列表，这个被大家广泛的使用，<ul> 还被更多的用于导航条
  • <ol>、<li> 有序列表
  • <dl>、<dt>、<dd> 就是“定义列表”，比如词典里面的词的解释、定义就可以用这种列表
  • <em> 是用作强调
  • <strong> 是用作强调
  • <table>、<tr>、<td>、<caption> 如果是为了标记列表的数据，就应该使用表格了。
    • <thead>为表格中的表头内容
    • <tbody>为表格中的主体内容
    • <tfoot>为表格中的表注内容（脚注）
    • <caption>为表格标题
    • <th>为表头单元格
    • <tr>为表格中的行
    • <td>为表格中的单元
  • <del>删除，知道<del>就不要再用 <s> 做删除线了，用 <del> 显然更具有语义化
  • <ins> 是表示插入
  • <abbr> 标签是表示 web 页面上的简称
  • <q> 来标记简短的单行引用
  • <blockquote> 块引用
  • <cite> 参考文献的引用

⇒模块化的目的

实现多人协同开发页面，提高页面研发速度和降低维护难度。研发速度的提升体现在多人协同并行开发， 维护难度体现在减少版本的混乱，根据模块区分版本降低版本间代码冲突和文件错误覆盖。

⇒拆分页面模块

一个页面有很多个小单元模块组成，他来自有原始需求文档，比如logo，导航，内容1，内容2，内容3，内容4，尾部导航，版权信息等等；根据他们就可以拆分出基本的模块。

⇒划分模块的原则

  • 哪些模块外观结构一样，我们就可以把它们归为一个模块
  • 模块的取名要符合语义化
  • 模块取名应加"mod-"前缀；如 .mod-property
  • 不要给header、mainNav、footer、pop等常用模块加"mod-"前缀
  • 编写模块样式时，选择器使用要具有独立性
  • 编写模块样式时，要先将公共样式提取编写；然后私有样式要用独立的class命名
  • 模块不应设置width和height值，要具有很好的适应性
  • 模块一率采用margin-bottom设置间隔

⇒伪类 和 伪元素

CSS3提倡伪类 用一个“:”链接选择符与伪类；例如 a:hover{…};

CSS3提倡伪元素 用两个“::”链接选择符与伪元素；例如 div::after{…};

⇒使用语义化的标签编写页面

⇒使用外部CSS和JS

⇒减少或删除注释 , 注释是给程序员看的

⇒减少表格 , 尤其是表格嵌套

⇒<h1> 标签每个页面只能出现一次 , <h1> 不要用在 LOGO 上

⇒<h1> 用在页面主体内容唯一的地方。如：单个文章列表页,文章或产品内容页

⇒<a> 标签必需加 title，<img /> 标签必需加 alt

⇒使用 <meta /> 标签

  1. <meta name="author" content="作者" />
  2. <meta name="keywords" content="关键词1|关键词2|关键词n" />
  3. <meta name="description" content="网页描述内容" />

⇒禁止对文字使用与背景色一样的文字颜色以达到隐藏的效果

⇒将 CSS 合并成一个文件 , 压缩代码

⇒尽可能的合并元素图片以减少 HTTP 请求，图片之间空隙尽量要小，图片文件也不宜太大 , 图片格式优先选用 PNG8 , 效果不佳可以使用 PNG24

⇒不要在 HTML 中缩放图像

⇒把样式表置于顶部

⇒避免使用 CSS表达式（Expression）

⇒用<link>代替 @import

⇒避免使用滤镜

⇒使用外部 CSS

⇒削减 CSS
